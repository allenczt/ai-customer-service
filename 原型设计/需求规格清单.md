# 大向智行 · AI智能客服需求规格清单

---

## 一、产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 大向智行 AI 智能客服 |
| 前台形态 | 微信小程序内置悬浮客服入口 |
| 后台入口 | 大向智行管理后台 → 智能客服（子模块） |
| 核心功能 | 基于AI大模型的智能问答系统，支持业务咨询、政策解读、流程指引等；后台支持问答管理、知识库文档上传、标签管理、会话记录查阅及客服设置 |
| 目标用户（前台） | 企业客户（安全培训、保险、银行借贷、车队管理、车后市场业务用户） |
| 管理用户（后台） | 运营人员 / 管理员 |

---

## 二、后台菜单结构

> 后台作为大向智行管理后台的**子模块**，一级入口菜单名称为「**智能客服**」。

```
大向智行管理后台
└── 智能客服（一级菜单）
    ├── 问题管理        ← 预设问答的增删改查
    ├── 知识库管理      ← 文档上传与管理，AI 学习用
    ├── 标签管理        ← 统一标签体系，问题和知识库通用
    ├── 会话记录        ← 用户提问历史查阅
    └── 客服设置        ← 客服热线 + 系统设置
```

## 六、接口要求

### 6.1 AI 对话接口

| 项目 | 要求 |
|------|------|
| 请求方式 | POST |
| 请求地址 | 由 AI 团队提供，在客服设置中配置 |
| 请求参数 | `message`（用户问题）、`tags`（相关标签数组）、`session_id`（会话ID） |
| 返回字段 | `answer`（AI 回答）、`resolved`（是否命中知识库）
| 响应时间 | ≤ 3 秒 |
| 错误处理 | 接口异常时返回默认引导语并提示转人工 |

### 6.2 知识库处理接口

| 项目 | 要求 |
|------|------|
| 功能 | 文档上传后触发解析，将内容向量化入库，供 AI 检索 |
| 处理时间 | 10MB 以内文档建议 ≤ 60 秒完成 |
| 状态回调 | 处理完成/失败时更新文档处理状态 |
| 支持格式 | doc/docx、pdf、md、ppt/pptx、xls/xlsx |

### 6.3 会话管理

| 项目 | 要求 |
|------|------|
| session_id | 每次打开客服生成唯一会话 ID，会话超时后失效 |
| 历史记录 | 会话记录持久化存储，后台可查阅 |

## 八、交互流程

### 8.1 前台（小程序端）· AI 客服咨询主流程

```
用户点击悬浮按钮打开聊天
        ↓
展示欢迎语 + 6大分类按钮（气泡内）+ 小象头像
        ↓
用户点击分类 → 显示为对话消息
        ↓
AI 回复该分类 Top5 常见问题 + 查看更多
        ↓
用户点击问题 → 显示为对话消息
        ↓
┌─────────────────────────────────────────────────────────────────┐
│  分支A: 问答库命中                                               │
│    直接返回标准答案 → 显示答案 → 问题解决 ✓                       │
├─────────────────────────────────────────────────────────────────┤
│  分支B: 问答库未命中                                             │
│    显示「正在检索知识库...」→ 综合知识库 + 大模型生成回答          │
│    → 用户满意？                                                  │
│      ├─ 满意 → 问题解决 ✓                                        │
│      └─ 不满意 → 继续咨询 / 转人工                               │
├─────────────────────────────────────────────────────────────────┤
│  分支C: 主动转人工                                               │
│    用户点击「人工客服」→ 转接人工客服 → 人工处理 → 会话结束 ✓     │
└─────────────────────────────────────────────────────────────────┘
```

**前台交互规则：**
- 所有操作（选择分类、点击问题）均作为用户消息显示在对话中
- 分类展示在欢迎语气泡内，而非独立底部区域
- 点击分类后展示该分类常见问题列表
- 点击问题触发AI回复，根据是否命中问答库决定响应方式
- 未命中时显示「正在检索知识库...」思考状态
- 每条AI回复末尾显示「已解决 / 未解决」满意度反馈按钮
- 用户关闭窗口后再次进入保留最近一次会话记录（本地 sessionStorage），服务端完整持久化存储
- 顶部导航栏始终显示小象头像 + 人工客服呼叫按钮

### 8.2 后台管理端 · 知识库维护流程

```
管理员登录后台管理系统
        ↓
进入「智能客服」→「知识库管理」
        ↓
┌──────────────────────────┬──────────────────────────┐
│  方式一: 上传文档         │  方式二: 手动添加          │
│  上传 PDF/Word/Excel/PPT  │  问答管理 → 添加问答       │
│  ↓                       │  ↓                        │
│  AI 自动解析 + 文本提取   │  填写: 问题/标准答案/分类  │
│  ↓                       │  ↓                        │
│  GraphRAG 索引构建        │  存入知识库数据库          │
└──────────────────────────┴──────────────────────────┘
                          ↓
              点击「立即同步」→ AI 知识库索引更新
                          ↓
              用户端实时生效 → 新问答可被检索
```

### 8.3 后台管理端 · 会话监控与数据分析流程

```
用户发起会话 → AI 回复
        ↓
系统实时记录: 用户问题 / AI 回复 / 耗时 / 状态
        ↓
┌────────────────────────────┬────────────────────────────┐
│  路径A: 会话记录 → 数据统计  │  路径B: 会话状态: 未解决/转人工 │
│  ↓                         │  ↓                          │
│  热门问题分析 → 优化知识库   │  人工介入 → 分析根因         │
│  ↓                         │  ↓                          │
│  数据大屏可视化展示          │  补充缺失问答 → 完善知识库    │
└────────────────────────────┴────────────────────────────┘
```

### 8.4 后台菜单结构

```
大向智行管理后台
└── 智能客服（一级菜单）
    ├── 首页仪表盘      ← 核心指标、热门问题TOP10、分类占比
    ├── 数据大屏        ← 30天趋势、时段分布、实时数据
    ├── 问答管理        ← 预设问答的增删改查
    ├── 知识库管理      ← 文档上传与管理，AI 学习用
    ├── 分类管理        ← 统一标签体系，问题和知识库通用
    ├── 会话记录        ← 用户提问历史查阅
    └── 客服设置        ← 客服热线 + 系统设置（头像/名称/欢迎语）
```
