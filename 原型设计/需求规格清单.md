# 大向智行 · AI智能客服需求规格清单

---

## 一、产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 大向智行 AI 智能客服 |
| 前台形态 | 微信小程序内置悬浮客服入口 |
| 后台入口 | 大向智行管理后台 → 智能客服（子模块） |
| 核心功能 | 基于AI大模型的智能问答系统，支持业务咨询、政策解读、流程指引等；后台支持问答管理、知识库文档上传、标签管理、会话记录查阅及客服设置 |
| 目标用户（前台） | 企业客户（安全培训、保险、银行借贷、车队管理、车后市场业务用户） |
| 管理用户（后台） | 运营人员 / 管理员 |

---

## 二、后台菜单结构

> 后台作为大向智行管理后台的**子模块**，一级入口菜单名称为「**智能客服**」。

```
大向智行管理后台
└── 智能客服（一级菜单）
    ├── 问题管理        ← 预设问答的增删改查
    ├── 知识库管理      ← 文档上传与管理，AI 学习用
    ├── 标签管理        ← 统一标签体系，问题和知识库通用
    ├── 会话记录        ← 用户提问历史查阅
    └── 客服设置        ← 客服热线 + 系统设置
```

### 3.2 知识库管理

> 用于上传业务文档，AI 在回答时会参考知识库内容进行检索增强（RAG）。

| 需求ID | 功能描述 | 优先级 | 备注 |
|--------|----------|--------|------|
| B20 | 文档列表展示 | P0 | 显示文件名、格式、大小、上传时间、关联标签、处理状态 |
| B21 | 上传文档 | P0 | 支持 `.doc/.docx`、`.pdf`、`.md`、`.ppt/.pptx`、`.xls/.xlsx` 格式，单文件最大 50MB |
| B22 | 批量上传 | P1 | 支持同时选择多个文件上传 |
| B23 | 文档处理状态 | P0 | 状态包括：处理中 / 已就绪 / 处理失败；处理完成后 AI 可检索 |
| B24 | 关联标签 | P1 | 文档上传时可绑定一个或多个标签，便于分类检索 |
| B25 | 删除文档 | P0 | 删除后 AI 不再检索该文档内容，需二次确认 |
| B26 | 文档预览/下载 | P2 | 支持在线预览（PDF/图片）或直接下载 |
| B27 | 重新处理 | P1 | 处理失败时支持手动重新触发解析 |
| B28 | 知识库同步 | P1 | 一键触发将最新文档同步至 AI 模型索引 |

**文档处理状态流转规则：**

| 状态 | 进入条件 | 离开条件 | 用户感知 |
|------|----------|----------|----------|
| `pending`（待处理） | 用户上传文档，文件接收成功 | 触发解析后进入 `processing` | 后台显示上传成功，等待处理 |
| `processing`（处理中） | 后端开始调用知识库处理接口解析文档 | 解析成功 → `ready`；解析失败 → `failed` | 后台显示进度或「处理中」标签 |
| `ready`（已就绪） | 文档解析完成，向量入库成功 | 文档被删除 → 消失 | 该文档内容可被 AI 检索 |
| `failed`（处理失败） | 解析过程报错或超时（>5分钟无响应） | 手动点击「重新处理」→「processing`」；或文档删除 | 后台显示红色失败标签 + 失败原因摘要 |

**知识库同步（B66）详细逻辑：**

| 项目 | 规则 |
|------|------|
| 触发方式 | 管理员手动点击「立即同步」按钮 |
| 同步内容 | 将所有 `status = ready` 的文档向量索引推送至 AI 模型，使新增/更新的文档可被检索 |
| 同步时间 | 建议在业务低峰期执行，避免影响 AI 响应速度 |
| 同步状态 | 显示「同步中…」（按钮禁用）+ 完成后显示「上次同步：YYYY-MM-DD HH:mm」 |
| 失败处理 | 同步失败时按钮恢复可点击，显示红色提示「同步失败，请重试」；不阻塞文档处理流程（文档处理完成即可被检索，仅同步失败时才不可检索） |

**支持文件格式：**

| 格式 | 扩展名 |
|------|--------|
| Word 文档 | `.doc` `.docx` |
| PDF | `.pdf` |
| Markdown | `.md` |
| PowerPoint | `.ppt` `.pptx` |
| Excel 表格 | `.xls` `.xlsx` |

### 3.4 会话记录

> 查看用户在小程序端提问的历史对话记录，用于质量追踪和问题优化。

| 需求ID | 功能描述 | 优先级 | 备注 |
|--------|----------|--------|------|
| B40 | 会话列表 | P0 | 显示会话ID、用户（脱敏）、首条问题摘要、状态、耗时、开始时间 |
| B41 | 会话详情 | P0 | 查看完整对话内容（用户问+AI答），按消息时间升序排列 |
| B42 | 状态筛选 | P1 | 支持按状态过滤：已解决 / 转人工 / 未解决 |
| B43 | 时间范围筛选 | P1 | 按会话开始时间的日期区间筛选会话列表 |
| B44 | 关键词搜索 | P1 | 支持对用户问题内容进行模糊搜索，**仅搜索用户消息（role = user）的 content 字段**，不搜索 AI 回复内容；匹配方式为包含匹配（LIKE '%keyword%'），不区分大小写；同时搜索会话列表和会话详情中的问题内容 |
| B45 | 导出记录 | P2 | 支持导出 Excel，含基础字段和对话摘要 |
| B46 | 用户信息脱敏 | P0 | 用户 ID 等敏感信息自动脱敏展示，如"用户***8" |

**会话列表字段定义：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 会话ID | 字符串 | 系统自动生成的唯一标识，格式为 `S` + 13位时间戳，如 `S1711871400000` |
| 用户（脱敏） | 字符串 | 用户微信OpenID经脱敏处理后展示，规则：保留首2位和末2位，中间替换为 `***`，如"用户***8C" |
| 首条问题摘要 | 字符串 | 取本次会话第一条用户消息内容，最多显示50个字符，超出截断显示省略号 |
| 状态 | 枚举 | 见下方「会话状态定义」 |
| 耗时 | 时长 | **计算规则**：`会话结束时间 - 会话开始时间`，单位秒，展示时格式化为 `mm分ss秒`（不足1分钟显示 `xx秒`）。若会话未结束（状态为进行中）则显示"进行中" |
| 开始时间 | 时间戳 | **代表用户本次会话第一条消息的发送时间**，格式：`YYYY-MM-DD HH:mm`，列表中按此字段降序排列（最新在前） |

**会话详情消息字段定义：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 消息时间 | 时间戳 | 每条消息的实际发送/接收时间，格式 `HH:mm:ss`；**用户消息**为用户点击发送的时刻，**AI消息**为AI回复完成并推送至前端的时刻 |
| 消息类型 | 枚举 | `user`（用户消息）/ `ai`（AI回复）/ `system`（系统通知，如会话超时提示） |
| 响应耗时 | 时长 | **仅AI消息有此字段**，计算规则：`AI消息时间 - 上一条用户消息时间`，单位毫秒，展示格式 `x.xs`，用于评估AI响应速度 |
| 命中类型 | 枚举 | **仅AI消息有此字段**：`qa_hit`（命中预设问答）/ `rag_hit`（命中知识库）/ `llm_fallback`（大模型兜底） |
| 满意度 | 枚举 | **仅AI消息有此字段**：`resolved`（已解决）/ `unresolved`（未解决）/ `null`（未反馈） |

**会话状态定义：**

| 状态 | 触发条件 | 说明 |
|------|----------|------|
| 进行中 | 会话创建后尚未触发任何结束条件 | `started_at` 已记录，`ended_at` 为 `null` |
| 已解决 | 用户在会话期间点击了任意 AI 回复的「已解决」按钮（满意度字段被置为 `resolved`）；**或**：会话超时自动结束时，最后一条 AI 消息的满意度字段为 `resolved` | 两者满足其一即触发，`ended_at` = 对应判定时刻 |
| 转人工 | 用户点击顶部「人工客服」按钮（无论是否实际拨通电话） | `ended_at` = 用户点击该按钮的时刻 |
| 未解决 | 会话超时自动结束，但最后一条 AI 消息的满意度字段不为 `resolved`；**或**：用户直接关闭小程序/切换至后台超过会话超时时间且无任何消息活动 | `ended_at` = 系统检测到超时或用户离开的时刻 |

> **注意**：「转人工」的判定以用户点击按钮为标准，不要求电话实际拨通或通话完成。

**会话结束决策树：**

```
用户打开客服窗口
        │
        ▼
   会话创建 → status = "active"，started_at = 第一条消息时间
        │
        ├─── 用户发送消息 ────→ AI 回复
        │                              │
        │    ┌─────────────────────────┼─────────────────────────┐
        │    │                         │                         │
        │    ▼                         ▼                         ▼
        │ 用户点击「已解决」    会话超时（无消息    用户点击「人工客服」
        │    │                  活动超过 N 分钟）   │
        │    ▼                         │                         ▼
        │ ended_at = 此时刻        检查最后一条AI消息   ended_at = 此时刻
        │ status = "resolved"        的满意度字段        status = "transferred"
        │                              │
        │                    ┌────────┴────────┐
        │                    ▼                 ▼
        │              满意度 = resolved   满意度 ≠ resolved
        │                    │                 │
        │                    ▼                 ▼
        │              status = resolved  status = unresolved
        │              ended_at = 此时刻   ended_at = 此时刻
        │
        └─── 用户关闭小程序/切换后台 ──────────────────────→ 后台定时任务检测
                                                          │
                                                  距最后消息 > N 分钟？
                                                          │
                                                ┌─────────┴─────────┐
                                                ▼                   ▼
                                             是（超时）          否（继续等待）
                                                │                   │
                                                ▼                   ▼
                                          status = "unresolved"   status = "active"
                                          ended_at = 此时刻       (不变)
```

**会话超时机制详细规则：**

| 项目 | 规则 |
|------|------|
| 超时判断对象 | 整个会话的最后一条消息（用户消息或 AI 消息）距离当前时刻的时长 |
| 超时时间来源 | 取后台「系统设置 → 会话超时时间」配置值（默认 60 分钟） |
| 超时检测方式 | 后端定时任务轮询，检测所有 `status = "active"` 的会话，距最后消息时间超过配置值则触发超时 |
| 超时触发动作 | 写入 `ended_at = 当前时刻`；根据最后 AI 消息满意度判定 `status`；向用户推送一条 `system` 类型消息：「您好，您的会话已超时，如需继续请重新发起咨询」 |
| 用户感知 | 用户下次打开客服窗口时，看到超时提示气泡（不自动弹出，由用户主动触发） |
| 消息计入 | 超时 `system` 消息计入 `message_count`；超时前的正常 AI 消息的 `response_ms` 仍计入平均响应时长 |

**耗时计算示意：**

```
用户第一条消息（10:00:00）← 会话开始时间
       ↓
AI 回复（10:00:03）        ← 响应耗时 = 3s
       ↓
用户追问（10:02:10）
       ↓
AI 回复（10:02:14）        ← 响应耗时 = 4s
       ↓
用户点击「已解决」（10:02:20）← 会话结束时间

会话耗时 = 10:02:20 - 10:00:00 = 2分20秒
平均响应耗时 = (3s + 4s) / 2 = 3.5s
```

**敏感词过滤拦截逻辑：**

| 项目 | 规则 |
|------|------|
| 触发时机 | 用户点击发送按钮后、后端正式处理前，前端也可做一次预检 |
| 匹配范围 | 用户输入的消息文本（`content` 字段）全文，包含标点符号 |
| 匹配方式 | 精确子串匹配（例：词库含「贷款」则「我想咨询贷款业务」触发拦截） |
| 拦截动作 | 阻止消息发送，弹出提示：「您的输入包含敏感词，请修改后重试」；消息不存入数据库，不计入会话消息数 |
| 敏感词管理 | 后台支持添加/编辑/删除敏感词；修改后实时生效（无需重启服务） |

**欢迎语触发规则：**

| 场景 | 触发条件 | 显示内容 |
|------|----------|----------|
| 首次打开 | 用户点击悬浮按钮进入客服（会话 `status = active` 且 `message_count = 0`） | 显示后台配置的欢迎语文本 + 小象头像 + 6大分类按钮气泡 |
| 会话中断后重连 | 用户关闭客服窗口后重新打开（`status = active` 但 `message_count > 0`） | **不重复显示欢迎语**，直接恢复上次会话记录（sessionStorage）并定位到上次位置 |
| 会话结束后重开 | 用户上一次会话已结束（`status = resolved/unresolved/transferred`），再次点击悬浮按钮 | 视为新会话，显示欢迎语 + 6大分类按钮气泡 |

**转人工逻辑：**

| 场景 | 行为 |
|------|------|

**AI 回答流程与问题匹配决策树：**

> 当用户发送一条消息（点击分类名称 / 点击预设问题 / 手动输入）时，后端按以下顺序依次检索：

```
用户发送消息（分类名称 / 预设问题 / 自由文本）
                │
                ▼
        【阶段一：精确匹配】
        在问答库（QA表）中检索：
        ├─ 点击分类名称 → 直接查该分类下的 Top5 问题列表，命中后返回标准答案
        ├─ 点击预设问题 → 按 qa_id 精确命中，直接返回答案
        └─ 自由输入 → 按语义相似度匹配：
                      标准问题 + 相似问法（共最多11条）逐一比对
                      相似度阈值由 AI 接口返回控制
                │
        ┌───────┴───────┐
        ▼               ▼
     命中（resolved）              未命中（unresolved）
        │                            │
        ▼                            ▼
  返回 qa_hit 标准答案       【阶段二：知识库检索（RAG）】
  前端展示答案               调用 GraphRAG 接口，在知识库文档中检索相关内容
  命中标签显示「问答命中」         │
                                ▼
                        命中内容片段？
                        ├─ 有 → 【阶段三：大模型综合】
                        │       将检索到的内容片段 + 用户问题 → AI 大模型
                        │       生成综合回答，前端命中标签显示「知识库」
                        │
                        └─ 无 → 【阶段四：大模型兜底】
                                仅凭用户问题 → AI 大模型直接回答
                                前端命中标签显示「大模型」
                                ⚠️ 回答末尾自动追加：「当前回答仅供参考，如有疑问请联系人工客服」
```

**各阶段返回内容格式：**

| 阶段 | 命中类型 | 前端显示 | 特殊处理 |
|------|----------|----------|----------|
| 阶段一（问答库精确匹配） | `qa_hit` | 标准答案（直接展示，无动画） | 命中后 `hit_count` +1 |
| 阶段二+三（知识库检索） | `rag_hit` | 答案 + 知识库思考状态动画（3秒内不消失，接口返回后替换） | 若 AI 接口超时（>10秒），显示「知识库检索超时，请稍后再试或联系人工客服」 |
| 阶段四（大模型兜底） | `llm_fallback` | 答案 + 末尾追加参考提示 | 仅在大模型无知识库检索结果时触发 |

**分类 Top5 问题列表逻辑：**

| 场景 | 行为 |
|------|------|
| 用户点击分类 | AI 回复该分类下 `status = active` 且 `hit_count` 最高的 5 条问答 + 末尾「查看更多」按钮 |
| 点击「查看更多」 | 切换到分类下全部问答列表（分页，每页10条），支持搜索 |

| 分类下不足5条 | 显示实际条目数，不显示「查看更多」 |

## 六、接口要求

### 6.1 AI 对话接口

| 项目 | 要求 |
|------|------|

### 6.4 核心数据字段定义

> 本节定义系统中所有关键字段的生成规则、格式约束与含义，供前后端开发对齐。

**会话（Session）字段：**

| 字段名 | 字段键 | 类型 | 生成规则 | 格式示例 |
|--------|--------|------|----------|----------|
| 会话ID | `session_id` | 字符串 | 前端打开客服窗口时生成：`S` + 当前时间戳（13位毫秒级） | `S1743465000123` |
| 会话开始时间 | `started_at` | ISO 8601 时间戳 | 用户本次会话第一条消息发送时刻 | `2026-04-01T09:00:00.000Z` |
| 会话结束时间 | `ended_at` | ISO 8601 时间戳 / null | 会话结束时写入：① 用户点击「已解决」时刻；② 用户点击转人工时刻；③ 系统检测超时时刻；④ 用户关闭窗口时刻。会话进行中为 `null` | `2026-04-01T09:05:00.000Z` |
| 会话耗时 | `duration_seconds` | 整数 | 后端计算：`ended_at - started_at`（秒），会话进行中为 `null` | `300` |
| 会话状态 | `status` | 枚举 | `active`（进行中）/ `resolved`（已解决）/ `transferred`（转人工）/ `unresolved`（未解决） | `resolved` |
| 消息数 | `message_count` | 整数 | 本次会话中所有消息（用户 + AI + 系统）的总条数 | `8` |

**消息（Message）字段：**

| 字段名 | 字段键 | 类型 | 说明 | 格式示例 |
|--------|--------|------|------|----------|
| 消息ID | `message_id` | 字符串 | `M` + session_id 末8位 + 自增序号（3位，从001起） | `M65000123_001` |
| 所属会话 | `session_id` | 字符串 | 关联的会话ID | `S1743465000123` |
| 消息类型 | `role` | 枚举 | `user`（用户发送）/ `ai`（AI回复）/ `system`（系统通知） | `user` |
| 消息内容 | `content` | 字符串 | 消息文本内容，最大长度 2000 字符 | `请问安全培训证书怎么办理？` |
| 发送时间 | `created_at` | ISO 8601 时间戳 | 用户消息：用户点击发送的时刻；AI 消息：AI 回复完成并推送至前端的时刻 | `2026-04-01T09:00:05.000Z` |
| AI响应耗时 | `response_ms` | 整数 / null | **仅 AI 消息**：`AI消息 created_at - 上一条 user 消息 created_at`（毫秒）；非 AI 消息为 `null` | `2340` |
| 命中类型 | `hit_type` | 枚举 / null | **仅 AI 消息**：`qa_hit` / `rag_hit` / `llm_fallback`；非 AI 消息为 `null` | `rag_hit` |
| 满意度 | `satisfaction` | 枚举 / null | **仅 AI 消息**：`resolved` / `unresolved` / `null`（未反馈） | `resolved` |
| 满意度反馈时间 | `feedback_at` | ISO 8601 时间戳 / null | 用户点击满意度按钮的时刻；未反馈为 `null` | `2026-04-01T09:05:00.000Z` |

**问答条目（QA）字段：**

| 字段名 | 字段键 | 类型 | 说明 |
|--------|--------|------|------|
| 问答ID | `qa_id` | 字符串 | `QA` + 6位自增数字，如 `QA000001` |
| 所属标签 | `tag_id` | 字符串 | 关联分类标签的ID |
| 标准问题 | `question` | 字符串 | 标准化的问题文本，最大200字 |
| 相似问法 | `similar_questions` | 字符串数组 | 同一问题的不同表述方式，用于语义匹配，最多10条 |
| 答案 | `answer` | 字符串 | 对应回答，支持 Markdown，最大5000字 |
| 状态 | `status` | 枚举 | `active`（已启用）/ `disabled`（已禁用）/ `draft`（草稿） |
| 命中次数 | `hit_count` | 整数 | 被用户问题匹配到的累计次数，每次命中自增1 |
| 创建时间 | `created_at` | ISO 8601 时间戳 | 条目创建时刻 |
| 最近更新 | `updated_at` | ISO 8601 时间戳 | 最近一次编辑保存的时刻 |

## 八、交互流程

### 8.1 前台（小程序端）· AI 客服咨询主流程

```
用户点击悬浮按钮打开聊天
        ↓
展示欢迎语 + 6大分类按钮（气泡内）+ 小象头像
        ↓
用户点击分类 → 显示为对话消息
        ↓
AI 回复该分类 Top5 常见问题 + 查看更多
        ↓
用户点击问题 → 显示为对话消息
        ↓
┌─────────────────────────────────────────────────────────────────┐
│  分支A: 问答库命中                                               │
│    直接返回标准答案 → 显示答案 → 问题解决 ✓                       │
├─────────────────────────────────────────────────────────────────┤
│  分支B: 问答库未命中                                             │
│    显示「正在检索知识库...」→ 综合知识库 + 大模型生成回答          │
│    → 用户满意？                                                  │
│      ├─ 满意 → 问题解决 ✓                                        │
│      └─ 不满意 → 继续咨询 / 转人工                               │
├─────────────────────────────────────────────────────────────────┤
│  分支C: 主动转人工                                               │
│    用户点击「人工客服」→ 转接人工客服 → 人工处理 → 会话结束 ✓     │
└─────────────────────────────────────────────────────────────────┘
```

**前台交互规则：**
- 所有操作（选择分类、点击问题）均作为用户消息显示在对话中
- 分类展示在欢迎语气泡内，而非独立底部区域
- 点击分类后展示该分类常见问题列表
- 点击问题触发AI回复，根据是否命中问答库决定响应方式
- 未命中时显示「正在检索知识库...」思考状态
- 每条AI回复末尾显示「已解决 / 未解决」满意度反馈按钮
- 用户关闭窗口后再次进入保留最近一次会话记录（本地 sessionStorage），服务端完整持久化存储
- 顶部导航栏始终显示小象头像 + 人工客服呼叫按钮

### 8.2 后台管理端 · 知识库维护流程

```
管理员登录后台管理系统
        ↓
进入「智能客服」→「知识库管理」
        ↓
┌──────────────────────────┬──────────────────────────┐
│  方式一: 上传文档         │  方式二: 手动添加          │
│  上传 PDF/Word/Excel/PPT  │  问答管理 → 添加问答       │
│  ↓                       │  ↓                        │
│  AI 自动解析 + 文本提取   │  填写: 问题/标准答案/分类  │
│  ↓                       │  ↓                        │
│  GraphRAG 索引构建        │  存入知识库数据库          │
└──────────────────────────┴──────────────────────────┘
                          ↓
              点击「立即同步」→ AI 知识库索引更新
                          ↓
              用户端实时生效 → 新问答可被检索
```

### 8.3 后台管理端 · 会话监控与数据分析流程

```
用户发起会话 → AI 回复
        ↓
系统实时记录: 用户问题 / AI 回复 / 耗时 / 状态
        ↓
┌────────────────────────────┬────────────────────────────┐
│  路径A: 会话记录 → 数据统计  │  路径B: 会话状态: 未解决/转人工 │
│  ↓                         │  ↓                          │
│  热门问题分析 → 优化知识库   │  人工介入 → 分析根因         │
│  ↓                         │  ↓                          │
│  数据大屏可视化展示          │  补充缺失问答 → 完善知识库    │
└────────────────────────────┴────────────────────────────┘
```

### 8.4 后台菜单结构

```
大向智行管理后台
└── 智能客服（一级菜单）
    ├── 首页仪表盘      ← 核心指标、热门问题TOP10、分类占比
    ├── 数据大屏        ← 30天趋势、时段分布、实时数据
    ├── 问答管理        ← 预设问答的增删改查
    ├── 知识库管理      ← 文档上传与管理，AI 学习用
    ├── 分类管理        ← 统一标签体系，问题和知识库通用
    ├── 会话记录        ← 用户提问历史查阅
    └── 客服设置        ← 客服热线 + 系统设置（头像/名称/欢迎语）
```
